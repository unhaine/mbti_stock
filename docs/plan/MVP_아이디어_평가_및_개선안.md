## 📌 구현 전략

### 핵심 원칙
```
✅ UI/UX: 전체 페이지 구조 완성 (앱 경험 완성도)
✅ 기능: 핵심만 구현 (온보딩 + MBTI 추천)
✅ 나머지: 목업/더미 데이터 (추후 구현 준비)
```

### 왜 이 전략인가?
1. **빠른 프로토타입** - 전체 흐름 체험 가능
2. **피드백 수집** - 실제 사용자 반응 확인
3. **점진적 구현** - 검증된 기능부터 추가
4. **포트폴리오 가치** - 완성도 높은 UI/UX 시연

## 🎯 최종 MVP 범위


### 구현할 것 (Phase 1)

#### 1️⃣ 페이지 구조 (전체 구현)
```
✅ 온보딩 페이지
   - MBTI 선택 화면
   - 로딩 화면

✅ 메인 앱 구조
   - 헤더 (로고, 알림, 설정 아이콘)
   - 메인 콘텐츠 영역
   - 푸터 메뉴바 (홈/커뮤니티/자산/설정)

✅ 홈 화면
   - MBTI 페르소나 카드
   - 좌우 스와이프 테마 영역
   - 종목 리스트

✅ 종목 상세 모달
   - 종목 정보 표시
   - 은유적 설명

✅ 커뮤니티 페이지 (껍데기)
   - "준비 중" 안내 화면

✅ 자산현황 페이지 (껍데기)
   - "준비 중" 안내 화면

✅ 설정 페이지
   - MBTI 변경 기능
   - 기본 설정 UI
```

#### 2️⃣ 핵심 기능 (실제 구현)
```
✅ 온보딩 플로우
   - MBTI 16개 유형 선택
   - 선택 상태 저장 (LocalStorage)
   - 로딩 애니메이션

✅ MBTI 기반 추천 시스템
   - MBTI → 투자 성향 매핑 (JSON)
   - 페르소나 생성 (룰 기반)
   - 테마별 주식 추천 (정적 데이터)

✅ 스와이프 인터랙션
   - 좌우 스와이프로 테마 전환
   - 부드러운 애니메이션

✅ 종목 상세 보기
   - 모달 열기/닫기
   - 종목 정보 표시
```

#### 3️⃣ 목업/더미 처리 (추후 구현)
```
⏸️ AI 캐릭터 생성
   → 사전 정의된 16개 캐릭터 사용

⏸️ 실시간 주가 데이터
   → 정적 JSON 데이터 사용

⏸️ 북마크 기능
   → UI만 구현, 기능은 alert()

⏸️ 커뮤니티
   → "준비 중입니다" 페이지

⏸️ 자산현황
   → "준비 중입니다" 페이지

⏸️ 알림
   → 아이콘만 표시, 클릭 시 alert()
```
