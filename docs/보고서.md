## 2. 페이지별 분석

### 2.1 랜딩 페이지 (/)

#### 현재 상태
- 배경 파티클 애니메이션 작동
- 화면 상단에 콘텐츠 없이 빈 공간만 표시
- 스크롤해야 메인 콘텐츠 표시됨

#### 🔴 문제점
1. **첫 화면이 비어 보임**: 스크롤 없이는 CTA 버튼 및 주요 콘텐츠 미표시
2. **Hero 섹션 위치**: 뷰포트 중앙에 위치하지 않음
3. **로딩/렌더링 지연**: 초기 로딩 시 콘텐츠가 늦게 나타남

#### 💡 개선 제안
```
1. Hero 섹션을 뷰포트 중앙에 배치 (min-h-screen + flex center)
2. 첫 화면에 타이틀, 부제목, CTA 버튼 즉시 표시
3. 스크롤 유도 애니메이션 추가 (아래 화살표)
4. 로딩 스켈레톤 또는 페이드인 애니메이션 강화
```

---

### 2.2 온보딩 페이지 (/onboarding)

#### 현재 상태
- ✅ MBTI 4x4 그리드 배치
- ✅ 그룹별 라벨 (분석가, 외교관, 관리자, 탐험가)
- ✅ 이모지 + MBTI 코드 + 닉네임 표시
- ✅ 선택 시 하이라이트 효과

#### ⚠️ 개선 필요
1. **선택 피드백 부족**: 선택된 MBTI가 어떤 것인지 불명확
2. **스크롤 필요**: 모바일에서 전체 그리드 보려면 스크롤 필요
3. **확인 버튼 가시성**: 하단 버튼이 고정되어 있지 않음

#### 💡 개선 제안
```
1. 선택된 MBTI 카드에 체크 아이콘 또는 강조 테두리 추가
2. 그리드 카드 크기 조절로 스크롤 최소화
3. 하단 버튼을 sticky로 고정
4. 선택 시 햅틱 피드백 (모바일)
```

### 2.3 메인 페이지 (/main)

#### 현재 상태
- ✅ 자산 카드 + MBTI 프로필 + 코멘트
- ✅ 테마 캐러셀 (좌우 네비게이션)
- ✅ 종목 리스트 (스크롤 영역 분리)
- ✅ 변동성/배당 표시
- ✅ 종목 클릭 → 전체 화면 상세

#### ⚠️ 개선 필요
1. **캐러셀 인디케이터**: 현재 위치 표시는 있으나 터치 스와이프 지원 없음
2. **종목 카드 밀도**: 정보가 많아 시각적 혼잡
3. **새로고침 기능**: Pull-to-refresh 미지원
4. **검색 기능**: 종목 검색 기능 없음

#### 💡 개선 제안
```
1. 터치 스와이프로 테마 변경 지원
2. 종목 카드 레이아웃 단순화 (가격/변동률만 강조)
3. Pull-to-refresh 제스처 추가
4. 상단에 종목 검색 버튼 추가
5. 종목별 상승/하락 배경색 그라데이션 적용
```

### 2.4 종목 상세 모달

#### 현재 상태
- ✅ 전체 화면 오버레이
- ✅ 뒤로가기 버튼 (좌측 상단)
- ✅ 종목 아이콘 + 가격 + 변동률
- ✅ MBTI 관점 은유
- ✅ 종목 정보 카드 (시가총액, 변동성, 배당, 업종)
- ✅ 태그 리스트
- ✅ 면책조항

#### ⚠️ 개선 필요
1. **차트 없음**: 가격 추이 차트 미제공
2. **액션 버튼 없음**: 매수/매도 버튼 없음
3. **공유 기능**: 우측 상단 공유 버튼 동작 미구현
4. **스크롤 영역**: 콘텐츠가 많으면 스크롤 필요하나 인디케이터 없음

#### 💡 개선 제안
```
1. 미니 차트 추가 (최근 7일 추이)
2. 하단에 매수/매도 버튼 고정 (Phase 2)
3. 공유 기능 구현 (Web Share API)
4. 스크롤 가능 시 스크롤 힌트 추가
```

### 2.5 포트폴리오 페이지 (/portfolio)

#### 현재 상태
- ✅ 원형 차트 (주식/현금 비율)
- ✅ 총 자산 + 변동률
- ✅ 현금/주식 분리 표시
- ✅ 보유 종목 탭
- ✅ 개별 종목 카드 (보유량, 현재가, 수익률)

#### ⚠️ 개선 필요
1. **거래 내역 탭**: 비어있음 ("거래 내역" 플레이스홀더)
2. **원형 차트**: 애니메이션 없음, 정적 표시
3. **종목 클릭**: 메인 페이지처럼 상세 모달 연동 필요
4. **정렬 옵션**: 수익률순, 금액순 정렬 없음

#### 💡 개선 제안
```
1. 거래 내역 탭 구현 (가상 거래 로그)
2. 원형 차트 진입 애니메이션 추가
3. 종목 클릭 시 상세 모달 열기
4. 정렬 드롭다운 추가
5. 수익률 색상 강조 (녹색/빨간색 배경 그라데이션)
```

### 2.8 설정 페이지 (/settings)

#### 현재 상태
- ✅ MBTI 프로필 표시 + 변경 버튼
- ✅ 토글 스위치 (알림, 다크모드, 효과음)
- ✅ 앱 정보 (버전, 빌드, 개발)

3. **피드백/리뷰**: 연결 링크 없음
4. **연락처/도움말**: 없음

#### 💡 개선 제안
```
1. 다크모드 토글 → 테마 선택 (다크/라이트/시스템)으로 변경
2. 효과음 구현 또는 토글 제거
3. 피드백 보내기 기능 (mailto 또는 폼)
4. 자주 묻는 질문(FAQ) 링크 추가
```

## 3. 공통 개선점

### 3.1 접근성 (Accessibility)

| 항목 | 현재 상태 | 개선 방안 |
|------|-----------|-----------|
| 색상 대비 | dark-400 텍스트가 어둡게 보임 | WCAG AA 기준 4.5:1 대비 확보 |
| 키보드 네비게이션 | Tab 이동 순서 불명확 | focus-visible 스타일 추가 |
| 스크린 리더 | aria-label 없음 | 모든 버튼에 aria-label 추가 |
| 터치 타겟 | 일부 버튼이 48px 미만 | 최소 44px × 44px 확보 |

### 3.2 성능 최적화

| 항목 | 현재 상태 | 개선 방안 |
|------|-----------|-----------|
| 이미지 | 사용 없음 | 필요시 WebP + lazy loading |
| 코드 스플리팅 | 미적용 | React.lazy로 페이지별 분리 |
| 메모이제이션 | useMemo 사용 중 | useCallback 추가 검토 |
| 번들 크기 | 적정 | framer-motion 트리쉐이킹 확인 |

### 3.3 에러 처리

| 항목 | 현재 상태 | 개선 방안 |
|------|-----------|-----------|
| 네트워크 오류 | 미처리 | Error Boundary 추가 |
| 빈 데이터 | 일부만 처리 | 모든 리스트에 Empty State |
| 로딩 상태 | 일부만 처리 | 모든 비동기 작업에 로딩 표시 |

### 3.4 UX 패턴

| 항목 | 현재 상태 | 개선 방안 |
|------|-----------|-----------|
| Pull-to-refresh | 미구현 | 모바일 필수 패턴 |
| 스켈레톤 로딩 | 미구현 | 콘텐츠 영역에 적용 |
| 토스트 메시지 | 없음 | 작업 완료/실패 알림 |
| 확인 모달 | settings만 적용 | 위험 작업에 확인 요청 |
