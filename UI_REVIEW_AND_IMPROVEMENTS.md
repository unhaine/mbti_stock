# UI/UX 개선 검토 보고서

## 1. 개요

현재 어플리케이션의 전반적인 레이아웃, 컴포넌트 크기, 패딩 및 테마 일관성을 검토하였습니다. 최근 "Light Theme" 전환과 "컴포넌트 소형화" 요청을 반영하여, 추가적인 개선이 필요한 부분들을 정리했습니다.

## 2. 공통 사항 (Global)

- **Safe Area 처리**: 상단 헤더(`pt-[4.5rem]` 등)와 하단 네비게이션바 패딩 처리가 페이지마다 미세하게 다름 (e.g., `pt-[3.5rem]` vs `pt-14`). 이를 `src/components/layout/Layout.jsx` 같은 래퍼 컴포넌트로 통일하거나 상수화하여 관리하는 것을 권장.
- **배경색 통일**: 일부 모달이나 카드에서 여전히 하드코딩된 다크 테마 배경(`bg-dark-800` 등)이 남아있을 수 있음. 디자인 시스템 토큰(`bg-background`, `bg-surface`) 사용을 강화해야 함.
- **스크롤바 숨김**: `scrollbar-hide` 유틸리티가 일부 스크롤 영역에 누락되어 미관을 해칠 수 있음.

## 3. 페이지별 상세 검토

### 3.1. 메인 페이지 (MainPage)

- **상산 자산 카드**: 높이가 다소 커서 화면 점유율이 높음. 패딩을 줄이고 폰트 크기를 조절하여 더 컴팩트하게 만들 필요가 있음.
- **테마 캐러셀**: 화면 하단에 고정되어 있는데, 모바일 환경에서 하단 네비게이션바와 시각적으로 충돌할 수 있음. 위치를 조정하거나 높이를 줄여야 함.
- **종목 리스트**: 리스트 아이템 간의 간격(`mb-4` 등)을 조금 더 좁혀서(`gap-3` 또는 `space-y-3`) 한 화면에 더 많은 정보를 보여주는 것이 좋음.

### 3.2. 커뮤니티 페이지 (CommunityPage)

- **게시글 카드 크기**: 개별 게시글 카드의 패딩(`p-5` 등)이 모바일에서 다소 넓음. `p-4` 정도로 줄이고, 폰트 사이즈도 본문은 `text-sm`으로 통일하는 것이 좋음.
- **필터 탭**: 상단 필터(MBTI 그룹, 탭) 영역이 두 줄을 차지하여 본문 영역을 침범함. 스크롤 가능한 단일 행으로 합치거나 높이를 줄이는 디자인 변경 고려.
- **작성 버튼**: 플로팅 버튼(FAB) 형태가 아니라 헤더에 포함되어 있는데, 접근성을 위해 하단 우측 FAB로 변경하는 것도 고려해볼 만함.

### 3.3. 포트폴리오 페이지 (PortfolioPage)

- **원형 차트 크기**: `size={100}`은 작은 화면에서 부담스러울 수 있음. `size={80}` 정도로 줄이고 옆의 텍스트와 균형을 맞추는 것을 제안.
- **리스트 아이템**: 보유 종목 리스트의 높이가 큼. 주식 앱처럼 촘촘한 리스트 형태(Dense List)로 변경하여 스크롤 부담을 줄일 필요 있음.
- **탭 네비게이션**: "보유 종목" / "거래 내역" 탭의 높이를 줄이고 스타일을 더 심플하게 변경 (`bg-secondary-100` 박스 스타일 -> 밑줄 스타일 등).

### 3.4. 설정 페이지 (SettingsPage)

- **카드 여백**: 설정 항목들 사이의 여백이나 내부 패딩이 넉넉한 편. 정보를 밀도 있게 보여주기 위해 패딩 축소 (`p-4` -> `p-3`).
- **섹션 간격**: 섹션 간의 마진(`mb-6`)을 `mb-4`로 줄여서 스크롤 없이 더 많은 메뉴를 볼 수 있게 조정.

### 3.5. 온보딩 & 랜딩 페이지

- **버튼 크기**: `size="lg"` 버튼들이 화면 하단을 꽉 채우는 경향이 있음. 좌우 여백을 주거나 높이를 `56px` -> `48px` 수준으로 줄이는 것 검토.
- **애니메이션**: 스크롤 스냅이 적용된 섹션들의 높이(`min-h-[80vh]`)를 컨텐츠 양에 맞춰 유동적으로(`min-h-fit` + 넉넉한 패딩) 바꾸는 것이 더 자연스러울 수 있음 (강제 스크롤 스냅은 호불호가 갈림).

## 4. 디자인 시스템 (Colors & Typography)

- **텍스트 계층**: `text-dark-50` (제목), `text-dark-300` (본문/보조) 외에 `text-dark-100`이나 `text-dark-200`의 사용 기준이 모호함. 3단계로 단순화하여 일관성 확보 필요.
- **그림자(Shadow)**: 리스트 아이템에 그림자가 너무 많으면 화면이 복잡해 보임. 중요 요소(카드, 버튼)에만 그림자를 쓰고 리스트는 보더(`border`)로 구분하는 "Flat"한 디자인으로 가는 방향성 제안.

## 5. 개선 우선순위 제안

1. **PortfolioPage & CommunityPage 컴팩트화**: 메인/설정 페이지는 수정되었으나 이 두 페이지는 아직 요소가 큼.
2. **MainPage 상단 영역 최적화**: 자산 정보와 테마 추천이 화면을 너무 많이 가리지 않도록 조정.
3. **전역 폰트 사이즈 조정**: 본문 기본 크기를 `16px` -> `15px` 또는 `14px`로 미세 조정하여 정보 밀도 향상.
